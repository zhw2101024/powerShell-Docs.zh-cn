- <span data-ttu-id="a209f-245">版本应为三个用句点隔开的数字块，如 `0.1.1` 或 `4.11.192`。</span><span class="sxs-lookup"><span data-stu-id="a209f-245">The version should be structured as three numeric blocks separated by periods, as in `0.1.1` or `4.11.192`.</span></span>
- <span data-ttu-id="a209f-246">以 `0` 开头的版本表明包尚无法用于生产，当 0 为唯一使用的数字时，第一个数字只应以 `0` 开头。</span><span class="sxs-lookup"><span data-stu-id="a209f-246">Versions starting with `0` indicate that the package isn't yet production ready, and the first number should only begin with `0` if that's the only number used.</span></span>
- <span data-ttu-id="a209f-247">第一个数字中的更改（例如，从 `1.9.9999` 更改为 `2.0.0`）表明为主要和重大版本更改。</span><span class="sxs-lookup"><span data-stu-id="a209f-247">Changes in the first number (`1.9.9999` to `2.0.0`) indicate major and breaking changes between the versions.</span></span>
- <span data-ttu-id="a209f-248">第二个数字块中的更改（例如，从 `1.1` 更改为 `1.2`）表明为功能级更改，如将新的 cmdlet 添加到模块。</span><span class="sxs-lookup"><span data-stu-id="a209f-248">Changes to the second number (`1.1` to `1.2`) indicate feature-level changes, such as adding new cmdlets to a module.</span></span>
- <span data-ttu-id="a209f-249">第三个数字块中的更改表明为非重大更改，如新参数、更新后的示例或新测试。</span><span class="sxs-lookup"><span data-stu-id="a209f-249">Changes to the third number indicate non-breaking changes, such as new parameters, updated samples, or new tests.</span></span>
- <span data-ttu-id="a209f-250">列出版本时，PowerShell 会将版本作为字符串进行排序，因此 `1.01.0` 将被视为大于 `1.001.0`。</span><span class="sxs-lookup"><span data-stu-id="a209f-250">When listing versions, PowerShell will sort the versions as strings, so `1.01.0` will be treated as greater than `1.001.0`.</span></span>

<span data-ttu-id="a209f-251">PowerShell 的创建时间先于 SemVer 发布时间，因此它支持大多数但非所有 SemVer 元素，具体而言：</span><span class="sxs-lookup"><span data-stu-id="a209f-251">PowerShell was created before SemVer was published, so it provides support for most but not all elements of SemVer, specifically:</span></span>

- <span data-ttu-id="a209f-252">它不支持在版本号中使用预发布字符串。</span><span class="sxs-lookup"><span data-stu-id="a209f-252">It doesn't support prerelease strings in version numbers.</span></span> <span data-ttu-id="a209f-253">如果发行者希望在提供版本 `1.0.0` 之后提供新的主要版本的预览版本，这就非常有用。</span><span class="sxs-lookup"><span data-stu-id="a209f-253">This is useful when a publisher wishes to deliver a preview release of a new major version after providing a version `1.0.0`.</span></span> <span data-ttu-id="a209f-254">今后发布的 PowerShell 库和 PowerShellGet  cmdlet 将支持这样做。</span><span class="sxs-lookup"><span data-stu-id="a209f-254">This will be supported in a future release of the PowerShell Gallery and **PowerShellGet** cmdlets.</span></span>
- <span data-ttu-id="a209f-255">PowerShell 和 PowerShell 库允许使用包含 1、2 和 4 段的版本字符串。</span><span class="sxs-lookup"><span data-stu-id="a209f-255">PowerShell and the PowerShell Gallery allow version strings with 1, 2, and 4 segments.</span></span> <span data-ttu-id="a209f-256">许多早期模块并未遵循这些指南，并且 Microsoft 产品版本在第 4 个数字块中包含生成信息（例如，`5.1.14393.1066`）。</span><span class="sxs-lookup"><span data-stu-id="a209f-256">Many early modules did not follow the guidelines, and product releases from Microsoft include build information as a 4th block of numbers (for example `5.1.14393.1066`).</span></span> <span data-ttu-id="a209f-257">从版本控制的角度来看，这些差异将被忽略。</span><span class="sxs-lookup"><span data-stu-id="a209f-257">From a versioning standpoint, these differences are ignored.</span></span>
